# AI Proxy — Implementation Plan

> Spec: `docs/superpowers/specs/2026-05-22-ai-proxy-design.md`

**Goal:** Ship the Cloudflare Worker proxy + client + deploy guide.
Plumbing only — AI features come later.

---

### Task 1: Cloudflare Worker

**Files:** Create `ai-proxy/worker.js`, `ai-proxy/wrangler.toml`

- [ ] `worker.js` (ES-module Worker):
  - CORS preflight + origin allowlist from `env.ALLOWED_ORIGINS`.
  - POST-only; parse `{ task, input }`.
  - `verifyFirebaseToken(token, projectId)` — RS256 verify against
    Google JWKS (`securetoken@system` JWK endpoint, module-cached);
    check `aud` / `iss` / `exp` / `sub`. 401 on failure.
  - `TASKS` map — `ping` only: minimal Claude call (system "reply
    pong", maxTokens 8).
  - `callClaude({system,user,maxTokens}, apiKey, model)` — POST
    Anthropic Messages API; return `{ ok, text }` or
    `{ error, status }`.
  - All responses carry CORS headers.
- [ ] `wrangler.toml` — name, main, compatibility_date, `[vars]`
  FIREBASE_PROJECT_ID / ALLOWED_ORIGINS / AI_MODEL (placeholders);
  note that ANTHROPIC_API_KEY is a secret.

### Task 2: Deploy guide

**Files:** Create `ai-proxy/README.md`

- [ ] Steps: Cloudflare account → `npm i -g wrangler` →
  `wrangler login` → fill `wrangler.toml` vars (Firebase project
  id, origins) → Anthropic key → `wrangler secret put
  ANTHROPIC_API_KEY` → `wrangler deploy` → copy URL to
  `VITE_AI_PROXY_URL` → set an Anthropic spend cap → smoke-test
  with the `ping` task (curl example).

### Task 3: Client

**Files:** Create `src/lib/aiClient.ts`; modify `.env.example`

- [ ] `isAIConfigured()` — `VITE_AI_PROXY_URL` non-empty.
- [ ] `callAI(task, input)` — get Firebase ID token, POST to the
  proxy with `Authorization: Bearer`, return `{ ok, text?, error? }`.
  Handle: not configured, no user, network error, non-2xx.
- [ ] `.env.example` — add `VITE_AI_PROXY_URL=` with a comment.

### Task 4: Test wiring + client test
